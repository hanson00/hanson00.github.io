加密算法#

在扣js的时候如果要扣的js代码不是很多可以全扣。（一万行代码以下，或者五千行代码以下）

在遇到js加密的情况下，如果该网站运用了加密算法，在该算法没有被魔改的情况下我们可以使用Python或JavaScript快速还原

摘要算法#

MD5#

MD5又称消息摘要算法，哈希算法，散列算法。密文长度是32位16进制数据 该算法的特征是：

  1. 单向加密，不可逆。明文经过算法加密后是不能反向推导明文，即：加密后不能解密。
  2. 密文长度是固定的。不论明文大小，经过相同的加密算法得到的密文长度是固定的。
  3. 密文通常以16进制的数据表示

在确认该算法前，还需判断算法有没有被魔改时：

  1. 调用网站加密函数，得到结果
  2. 调用一些在线算法加密网站拿相同的值去对比，模拟对比
  3. 明文一样，加密结果一样

SHA#

SHA又称安全哈希算法，主要适用于数字签名，SHA通常指的是SHA家族的五个算法：SHA-1、SHA-224、SHA-256、SHA-384、SHA-512。SHA是比MD5更安全一些的算法，SHA算法后面的版本数字越大越安全，代价是速度更慢

HMAC#

HMAC全称散列消息认证码、密钥相关的哈希运算消息认证码，HMAC加密算法是一种安全的基于加密Hash函数和共享密钥的消息认证协议。要求通信双方共享密钥Key、约定算法。然后对报文进行Hash运算，形成固定长度的认证码。通信双方通过认证码来确定报文的合法性。

形成的密文主要和：Key和明文有关

Key分两种：

  1. 在前端写死的
  2. 由后端接口返回的

import hmac

def hmac_test1():
    message = b'I love python!'
    key = b'secret'
    md5 = hmac.new(key, message, digestmod='MD5')
    print(md5.hexdigest())


def hmac_test2():
    key = 'secret'.encode('utf8')
    sha1 = hmac.new(key, digestmod='sha1')
    sha1.update('I love '.encode('utf8'))
    sha1.update('Python!'.encode('utf8'))
    print(sha1.hexdigest())

if __name__ == '__main__':
    hmac_test1()  # 9c503a1f852edcc3526ea56976c38edf
    hmac_test2()  # 2d8449a4292d4bbeed99ce9ea570880d6e19b61a
