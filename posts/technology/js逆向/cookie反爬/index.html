Cookie反爬虫#

前言：一般早期反爬虫是表单中的签名认证，但是过于简单容易被发现。所以放到cookie中不易被发现。

cookie反爬虫可以归类到参数验证

cookie反爬可以分为两种：

  1. 不需要分析js型：第一次访问服务器，服务器返回一个cookie值，第二次访问带着cookie值去访问服务器。httponly✔
  2. 需要分析js型：第一次访问服务器，服务器返回一个cookie值，然后浏览器用js对cookie进行加密，然后带着加密后的cookie访问服务器。

不需要分析js型-项目实战#

地址：http://www.zjmazhang.gov.cn/hdjlpt/published?via=pc

经过测试后发现反爬目标：

  1. Cookie，后端返回
  2. 请求头部参数X-CSRF-TOKEN：请求头中的这个参数，经过观察后可以发现是由后端返回到前端主页中的值

X-CSRF-TOKEN作用：后端返回的，防止请求伪造

#!/usr/bin/env python
# coding:utf-8
# @Time : 2023/3/26 11:06
# @Author : 糕阳
# @file : 不是session版本.py
# $software : PyCharm
import requests
import re

token_reg = re.compile("var _CSRF = '(?P<token>.*?)';")

headers = {
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/107.0.0.0 Safari/537.36',
    # 'X-CSRF-TOKEN': 'ZN3K1Qzjbe0lXz9V5cN60coh9YzupbfhBMrKU8ij',   防止请求伪造
}

def get_cookie_token():
    """
    访问首页得到cookie和token
    :return: cookie和token
    """
    url = 'http://www.zjmazhang.gov.cn/hdjlpt/published?via=pc'
    response = requests.get(url=url, headers=headers)
    token1 = token_reg.search(response.text).group('token')
    response.close()
    # print(token1)
    return response.cookies, token1

def get_data():
    cookies, token = get_cookie_token()
    headers['X-CSRF-TOKEN'] = token
    data = {
        'offset': '20',
        'limit': '20',
        'site_id': '759010',
    }
    url = 'http://www.zjmazhang.gov.cn/hdjlpt/letter/pubList'
    response = requests.post(url=url, data=data, cookies=cookies, headers=headers)
    print(response.json())
    response.close()

if __name__ == '__main__':
    get_data()

#!/usr/bin/env python
# coding:utf-8
# @Time : 2023/3/26 11:31
# @Author : 糕阳
# @file : session处理cookie反爬.py
# $software : PyCharm
import requests
import re

token_reg = re.compile("var _CSRF = '(?P<token>.*?)';")

def get_data():
    url = 'http://www.zjmazhang.gov.cn/hdjlpt/published?via=pc'
    headers = {
        'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/107.0.0.0 Safari/537.36',
        # 'X-CSRF-TOKEN': 'ZN3K1Qzjbe0lXz9V5cN60coh9YzupbfhBMrKU8ij',   防止请求伪造
    }
    session = requests.session()
    response = session.get(url=url, headers=headers)
    token1 = token_reg.search(response.text).group('token')
    headers['X-CSRF-TOKEN'] = token1
    data = {
        'offset': '20',
        'limit': '20',
        'site_id': '759010',
    }
    url = 'http://www.zjmazhang.gov.cn/hdjlpt/letter/pubList'
    # print(session.cookies)
    response2 = session.post(url=url, data=data, headers=headers)
    print(response2.json())
    response.close()
    response2.close()

if __name__ == '__main__':
    get_data()

#!/usr/bin/env python
# coding:utf-8
# @Time : 2023/3/26 15:41
# @Author : 糕阳
# @file : 请求.py
# $software : PyCharm
import requests
import re
import execjs

arg1_reg = re.compile("var arg1='(?P<arg1>.*?)';")

url = 'https://sh.esfxiaoqu.zhuge.com/page3/'
headers = {
    # 'Cookie':'acw_tc=2760778816798164025591901e629c3e38356291840f0b525fd6d8963ba96d;',
    'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/107.0.0.0 Safari/537.36',
}

response = requests.get(url=url, headers=headers)
# print(response.cookies)     # 观察下第一次返回的cookie信息

arg1 = arg1_reg.search(response.text).group('arg1')
ctil = execjs.compile(open('./核心加密算法.js', 'r', encoding='utf-8').read())
arg2 = ctil.call('run', arg1)
# 添加更新cookie
response.cookies.update({'acw_sc__v2':arg2})
# print(arg2)
# 赋值cookie使用
cookies = response.cookies

response2 = requests.get(url=url, headers=headers, cookies=cookies)
print(response2.text)
response2.close()
response.close()

马蜂窝#

加速乐(jsl)cookie反爬虫特征：

  1. cookie特征：__jsluid_s、__jsluid_s、__jsl_clearance_s
  2. 第一次访问时，会出现两个521(目的是后端返回cookie)，然后加密后再生成一个加密cookie。走三次请求

逆向分析：

  1. 第一次，直接请求后观察返回数据

返回的是生成cookie的代码

接下来我们对加密js文件进行算法还原

下图是第二次请求返回的js文件

接着插入debugger语句并替换该文件，进行调试还原加密算法(扣算法)

逆向总结：

  • 会走三次请求，第一次返回一个eval的JavaScript文件用于生成一个cookie，携带着第一次的cookie去请求一个新的JavaScript文件用于生成新的cookie，最后拿着第二次cookie获取最终的数据文件
  • 第二次获取的JavaScript文件，使用ast解混淆后可以方便的扣取加密逻辑，需要替换的是go参数
