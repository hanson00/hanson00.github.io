迭代器与生成器#

Python中的可迭代对象iterable：

  • list
  • dict
  • set
  • str
  • tuple

笼统的说法可迭代对象就是可以被for循环执行的对象

下面的代码—判断对象是不是一个可迭代对象

from collections.abc import Iterable
list1 = [1, 2, 3]
print(isinstance(list1, Iterable))

list类是通过一种反射机制继承了Iterable类。只要是Iterable类的子类就是一个可迭代对象

迭代对象：在实例对象对应的类中如果存在__iter__则这个实例是一个可迭代对象

迭代器对象：在实例对象对应的类中如果存在__next__则这个实例是一个迭代器对象

**迭代器对象的作用：可以在__next__中制定迭代规则。**下面有代码实例

迭代器是Python中的一种对象

nums = [1, 2, 3]
obj = iter(nums)
print(obj)  #输出 <list_iterator object at 0x000002C47D052B20>
print(obj.__next__())  #输出 1
print(obj.__next__())  #输出 2
print(next(obj))  # 推荐使用
  1. iter函数：传递一个可迭代对象，将这个可迭代对象转化为迭代器对象
  2. 可迭代对象不一定是迭代器对象，但迭代器对象一定是可迭代对象
  3. 迭代器对象中内置了__next_()方法，可以通过是否有该方法判断该对象是迭代器对象还是可迭代对象。
  4. 通过from collection.abc import Iterator通过print(isinstance(list1, Iterator))判断

迭代器对象的__next__方法的主要作用是返回迭代器的下一个元素。当迭代器的所有元素都被访问过后，再次调用__next__方法会引发一个StopIteration异常，这标志着迭代的结束。

当你创建一个迭代器对象时，Python会在其内部维护一个指向当前元素的位置。每次调用__next__方法时，迭代器会移动到下一个元素，并返回该元素的值。当没有更多的元素可以返回时，就会抛出StopIteration异常。

Python中建议开发者不要直接调用魔术方法，而是使用next(obj)。

class ListNode:
    def __init__(self):
        self.index = 0
        self.lists = list()

    def __iter__(self):
        return self

    def __next__(self):
        if self.index < len(self.lists):
            target = self.lists[self.index]
            self.index += 1
            return target
        else:
            print('元素被取完了')
            raise StopIteration

    def add(self, name):
        self.lists.append(name)


node1 = ListNode()
node1.add('anna')
node1.add('hanson')
for item in node1:
    print(item)

class StudentData:
    def __init__(self):
        self.index = 0
        self.stu_list = list()

    def add_stu(self, name, age):
        dict_stu = dict()
        dict_stu['name'] = name
        dict_stu['age'] = age
        self.stu_list.append(dict_stu)

    # 通过该方法将实例对象转为可迭代对象
    def __iter__(self):
        return self

    def __next__(self):
        if self.index < len(self.stu_list):
            stu = self.stu_list[self.index]
            self.index += 1
            return stu
        else:
            raise StopIteration


student_data = StudentData()
student_data.add_stu('anna', 22)
student_data.add_stu('hanson', 25)

for item in student_data:
    print(item['name'])

class StudentData:
    def __init__(self):
        self.index = 0
        self.stu_list = []

    def add_stu(self, name, age):
        self.stu_list.append({'name': name, 'age': age})

    def __iter__(self):
        if not self.stu_list:
            raise ValueError("No students added to StudentData, cannot iterate.")
        return self

    def __next__(self):
        while self.index < len(self.stu_list):
            stu = self.stu_list[self.index]
            self.index += 1
            if stu['age'] > 24:
                return stu
        raise StopIteration


student_data = StudentData()
student_data.add_stu('anna', 22)
student_data.add_stu('hanson', 25)
student_data.add_stu('joker', 30)

# 迭代学生数据并打印年龄大于24岁的学生名字
for stu in student_data:
    if stu is not None:
        print(stu['name'])

生成器#

生成器是一种特殊的迭代器，可以帮助我们完成并发任务（协程）。

在函数中有yield就会变成一个特殊的函数（生成器函数）

优点：

  1. 内存高效：爬虫经常需要处理大量数据，如果一次性加载所有数据到内存中，可能会导致内存不足。生成器按需生成数据，一次只处理一小部分数据，可以极大地减少内存使用，使得处理大规模数据时更加高效。
  2. 流式处理：生成器允许进行流式处理，即数据可以一边产生一边处理，而不需要等待所有数据都生成完毕。这对于处理网络爬虫获取的数据特别有用，因为网络请求可能会很慢，或者数据量可能非常大，流式处理可以减少等待时间并降低内存压力。
  3. 懒加载机制、简化代码：生成器可以帮助简化代码结构，使得代码更加清晰易读。通过定义生成器函数，你可以将复杂的数据获取和处理逻辑封装在函数内部，然后在需要的时候通过迭代来获取数据，而不需要在外部维护复杂的逻辑。
  4. 异步和并发：生成器与异步IO和并发编程可以很好地结合。在爬虫中，经常需要并发地发送多个请求，然后异步地处理响应。生成器可以与异步库（如asyncio）一起使用，实现高效的并发爬取。
  5. 易于扩展：生成器具有模块化和可复用的特点，这使得在复杂的爬虫项目中可以很容易地扩展和修改数据获取和处理逻辑。你可以创建多个生成器来处理不同的数据源或数据格式，然后在一个统一的框架中组合它们。
  6. 错误处理：生成器在处理数据时，可以方便地添加错误处理逻辑，比如跳过错误的数据项或重新尝试获取数据。这有助于提高爬虫的健壮性和稳定性。

def get_num(num):
    print('开始迭代')
    index = 0
    while index < num:
        print('迭代中')
        yield index
        print('迭代结束')
        index += 1

# 生成器函数不能直接的调用函数，直接调用是生成一个生成器对象
obj = get_num(3)
# print(obj)  #输出 <generator object get_num at 0x000001C1146BBF90>
print(obj.__next__())
print(obj.__next__())
print(next(obj))

生成器send()传递信号#

send()运行时必须先激活生成器，即该生成器会运行到yield

send()方法的使用方式如下：

  1. 生成器必须首先被启动，这通常是通过调用next()方法或send(None)来完成的。这将执行生成器函数直到遇到第一个yield语句，并返回该yield语句后面的值。
  2. 在生成器启动后，你可以使用send()方法向生成器发送一个值。这个值会被赋值给yield表达式的结果，然后生成器会继续执行直到下一个yield语句，并返回该yield语句后面的值。
  3. 如果生成器函数执行完毕（即没有更多的yield语句），再次调用send()方法会抛出StopIteration异常。

        # 第一次先返回yield后面的值，然后中断，此时并没有发生data赋值
        data = yield i
        if data == '我自己传值':
            print('密码6666')
        print('data变量的值：', data)
        i += 1


obj = get_num(5)
# 第一次先返回yield后面的值，然后中断，此时并没有发生data赋值
print(next(obj))

import asyncio
import random
import time


# 定义一个模拟异步任务的生成器函数
async def async_task(task_id):
    print(f"开始任务 {task_id}")
    await asyncio.sleep(random.random())  # 模拟异步IO操作，比如网络请求  
    print(f"结束任务 {task_id}")
    return task_id * 2  # 假设每个任务返回一个结果  


# 创建一个生成器，用于产生任务
def task_generator():
    task_id = 0
    while True:
        task_id += 1
        yield async_task(task_id)

    # 使用asyncio并发执行生成器中的任务


async def main():
    loop = asyncio.get_running_loop()
    task_gen = task_generator()

    # 创建任务列表  
    tasks = []
    for _ in range(5):  # 例如，并发执行5个任务  
        task = asyncio.create_task(next(task_gen))
        tasks.append(task)

        # 等待所有任务完成
    await asyncio.wait(tasks)

    # 收集并打印所有任务的结果  
    results = [task.result() for task in tasks]
    print("所有任务完成，结果如下：", results)


# 运行主协程
asyncio.run(main())
