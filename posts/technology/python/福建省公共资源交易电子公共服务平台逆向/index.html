目标网站#

https://ggzyfw.fj.gov.cn/business/list/

首先观察到源代码中没有要包含的数据，所以采用抓包观察

逆向思路和步骤#

  1. 打开抓包工具后进行关键词抓包定位

发现找不到关键词，发现数据是密文数据

如果是没有混淆的js有三种调式方式：

  • 直接搜索关键字decrypt
  • ajax渲染，搜索关键字JSON.parse
  • 跟着堆栈，进行调试直到明文数据
  1. 找到加密js位置

首先使用搜索关键字decrypt，发现有多个js，接着改用路径搜索

在找到该js后打开至源面板搜索decrypt

然后打上断点，并在函数返回值处也打上断点，并点击下一页让该加密逻辑重新执行，我们重新观察

然后把return返回的数据在控制台中打印观察是否为明文数据，结果为明文数据

接着往前推，把b函数（前推的函数）复制下来，并把b函数的参数扣出来

运行后根据报错一步步添加、修改

安装算法库为npm install crypto-js,然后对比与浏览器中的加密包

对比发现是无误后直接进行代码替换接着不断调试

调试后遇到错误后不断去浏览器中观察报错位置

不断调试后解密出数据

const Crypto = require('crypto-js')

function b(t) {
    var e = Crypto.enc.Utf8.parse("BE45D593014E4A4EB4449737660876CE")
        , n = Crypto.enc.Utf8.parse("A8909931867B0425")
        , a = Crypto.AES.decrypt(t, e, {
        iv: n,
        mode: Crypto.mode.CBC,
        padding: Crypto.pad.Pkcs7
    });
    return a.toString(Crypto.enc.Utf8)
}
